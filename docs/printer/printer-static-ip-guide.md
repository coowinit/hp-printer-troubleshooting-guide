# 📌 网络打印机固定 IP 与 DHCP 保留配置建议

![Platform](https://img.shields.io/badge/Platform-Network-blue)
![Topic](https://img.shields.io/badge/Topic-Static%20IP%20%2F%20DHCP-green)
![Status](https://img.shields.io/badge/Status-Guide-success)

> 本文用于说明为什么网络打印机需要稳定的 IP 地址，以及如何在办公室和企业环境中选择“手动固定 IP”或“DHCP 保留”，避免因打印机 IP 变化导致无法打印。

---

## 📌 适用场景

当出现以下情况时，建议检查打印机 IP 配置：

- 打印机之前能打印，后来突然不能打印
- 电脑端端口仍指向旧 IP
- Ping 原 IP 不通
- 重新添加打印机后发现 IP 变了
- 更换路由器后打印机无法打印
- 多台电脑连接同一台打印机，但部分电脑失效
- 打印机使用 TCP/IP 端口，但 IP 是自动获取的

---

## 🧠 为什么打印机需要固定 IP

网络打印机通常通过 IP 地址接收电脑发送的打印任务。

如果电脑端配置的是：

```text
IP_192.168.0.193
```

那么电脑会一直把任务发送到：

```text
192.168.0.193
```

如果打印机后来变成：

```text
192.168.0.205
```

电脑仍然会向旧 IP 发送任务，结果就是：

```text
打印任务进入队列
        ↓
电脑找不到原来的打印机
        ↓
打印失败或卡住
```

因此，TCP/IP 端口必须配合稳定的打印机 IP，才能长期可靠。

---

## ⚖️ 三种 IP 配置方式对比

| 配置方式 | 说明 | 优点 | 缺点 | 推荐程度 |
|---|---|---|---|---|
| 自动获取 IP | 打印机从路由器自动获取地址 | 简单 | IP 可能变化 | 不推荐长期使用 |
| 手动固定 IP | 在打印机上手动填写 IP | 直观、稳定 | 需要避免 IP 冲突 | 推荐 |
| DHCP 保留 | 在路由器 / 网关中绑定 MAC 与 IP | 集中管理、不易冲突 | 需要管理路由器 | 最推荐 |

---

## ✅ 推荐方案

企业环境最推荐：

```text
DHCP 保留
    ↓
打印机始终获得同一个 IP
    ↓
电脑使用 Standard TCP/IP Port
    ↓
所有电脑统一连接同一个 IP
```

小型办公室也可以使用：

```text
打印机手动固定 IP
    ↓
电脑使用 TCP/IP 端口
```

但需要注意避免 IP 冲突。

## 🔍 如何查看打印机当前 IP

常见方法：

### 方法一：打印网络配置页

很多网络打印机可以从面板中打印网络配置页，里面通常包含：

```text
IP Address
Subnet Mask
Gateway
MAC Address
Host Name
```

### 方法二：查看打印机屏幕

部分打印机可以在菜单中查看：

```text
Network
TCP/IP
IPv4 Address
```

### 方法三：查看路由器 / DHCP 客户端列表

在路由器或网关管理后台查找：

```text
设备名称
MAC 地址
当前 IP
连接方式
```

### 方法四：通过电脑 Ping 测试

```cmd
ping 192.168.0.193
```

如果返回：

```text
Reply from 192.168.0.193
```

说明该 IP 当前可以连通。

## 🛠️ DHCP 保留配置思路

DHCP 保留通常在路由器、防火墙或网关设备中设置。

基本逻辑：

```text
打印机 MAC 地址
        ↓
绑定一个固定 IP
        ↓
打印机每次自动获取时都拿到同一个 IP
```

建议记录：

| 项目 | 示例 |
|---|---|
| 设备名称 | HP-M1216-Office |
| MAC 地址 | 00:11:22:33:44:55 |
| 保留 IP | 192.168.0.193 |
| 网关 | 192.168.0.1 |
| 子网掩码 | 255.255.255.0 |
| 位置 | 前台 / 办公区 |

---

## 🛠️ 手动固定 IP 配置思路

如果无法访问路由器后台，也可以在打印机面板或打印机 Web 管理页面中设置手动 IP。

通常需要填写：

```text
IP Address：192.168.0.193
Subnet Mask：255.255.255.0
Default Gateway：192.168.0.1
DNS Server：可选
```

建议选择不容易冲突的 IP，例如：

```text
DHCP 自动分配范围：192.168.0.100 - 192.168.0.200
打印机固定 IP：192.168.0.50
```

或者在路由器中把打印机 IP 排除在自动分配范围之外。

---

## 🚨 IP 冲突的常见表现

如果打印机 IP 与其他设备冲突，可能出现：

- Ping 一会儿通、一会儿不通
- 打印机时好时坏
- 打印任务偶尔成功、偶尔失败
- 电脑访问打印机后台时打开的是其他设备
- 多台设备显示相同 IP
- 路由器日志出现 IP 冲突提示

处理建议：

```text
1. 确认打印机 MAC 地址
2. 查看路由器 DHCP 列表
3. 更换一个未使用的 IP
4. 设置 DHCP 保留
5. 重启打印机和电脑后重新测试
```

---

## 🔧 IP 变化后的处理流程

如果打印机 IP 已经变化，可以按下面处理：

```text
查看打印机新 IP
        ↓
Ping 新 IP 是否正常
        ↓
打开打印机属性
        ↓
进入“端口”选项卡
        ↓
修改旧 TCP/IP 端口或添加新端口
        ↓
勾选新端口
        ↓
打印测试页
```

示例：

```text
旧端口：IP_192.168.0.193
新 IP：192.168.0.205
处理：新增 IP_192.168.0.205 或修改端口地址
```

---

## 🧪 配置完成后的验证

建议完成固定 IP 或 DHCP 保留后，执行以下验证：

```text
1. 重启打印机
2. 等待打印机重新联网
3. 查看打印机 IP 是否保持不变
4. 在电脑上 Ping 打印机 IP
5. 打印 Windows 测试页
6. 重启电脑后再次打印
7. 第二天再次确认 IP 是否保持一致
```

如果多次重启后 IP 都不变，说明配置基本稳定。

## 🛠️ 常见问题速查

| 现象 | 可能原因 | 建议处理 |
|---|---|---|
| 旧电脑能打印，新电脑不能打印 | 新电脑端口配置错误 | 检查端口 IP |
| 所有电脑突然不能打印 | 打印机 IP 变化或网络异常 | 查看当前 IP 并 Ping 测试 |
| Ping 旧 IP 不通 | 打印机 IP 已变化或离线 | 查看新 IP / 检查网络 |
| Ping 新 IP 通但不打印 | 电脑仍使用旧端口 | 修改或新增 TCP/IP 端口 |
| 打印时好时坏 | IP 冲突或网络不稳定 | 设置 DHCP 保留并排查冲突 |

---

## 📝 总结

```text
网络打印机稳定性的关键：IP 不要变。
```

推荐优先级：

```text
DHCP 保留 > 手动固定 IP > 自动获取 IP
```

对于企业办公环境，建议统一采用：

```text
DHCP 保留 + Standard TCP/IP Port + 官方驱动 + 设备登记表
```

这样后续无论是新增电脑、排查故障，还是更换网络设备，都更容易维护。
