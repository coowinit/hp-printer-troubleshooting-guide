# 共享打印机故障排查指南

> 适用于 Windows 办公环境中通过某一台电脑共享打印机的场景，帮助判断共享主机、网络访问、权限、驱动和防火墙等常见问题。

---

## 1. 什么是共享打印机

共享打印机通常是指：

```text
打印机连接到一台电脑
        ↓
这台电脑开启打印机共享
        ↓
其他电脑通过网络访问这台电脑上的共享打印机
```

它和网络打印机的区别是：

| 类型 | 连接方式 | 是否依赖某台电脑 | 稳定性 | 推荐场景 |
|---|---|---|---|---|
| 共享打印机 | 通过共享主机转发 | 是 | 中等 | 小办公室、临时共享 |
| 网络打印机 | 打印机直接接入局域网 | 否 | 较高 | 企业环境、多人使用 |

> 企业环境更推荐“网络打印机 + 固定 IP / DHCP 保留 + TCP/IP 端口”。共享打印机适合临时、低成本或少量用户场景。

---

## 2. 常见故障现象

- 其他电脑突然无法打印
- 显示“无法连接到打印机”
- 显示“访问被拒绝”
- 打印机显示脱机
- 主机电脑能打印，其他电脑不能打印
- 共享打印机重新连接后提示需要驱动
- 主机重启后客户端打印恢复
- 客户端能看到共享打印机，但任务一直卡在队列中

---

## 3. 排查流程总览

```text
共享打印机无法使用
        ↓
确认共享主机是否开机
        ↓
确认共享主机本机能否打印
        ↓
确认客户端能否访问共享主机
        ↓
检查共享权限和网络发现
        ↓
检查防火墙和凭据
        ↓
检查驱动是否匹配
        ↓
重新连接共享打印机
```

---

## 4. 第一步：确认共享主机是否开机

共享打印机依赖共享主机电脑。

如果共享主机：

```text
关机 / 睡眠 / 断网 / 重启中
```

其他电脑通常无法打印。

建议检查：

- 主机电脑是否开机
- 主机是否进入睡眠
- 主机是否连接网络
- 主机是否更换了电脑名称或 IP
- 主机是否能正常打印测试页

---

## 5. 第二步：确认共享主机本机能否打印

在共享主机上直接打印测试页。

路径：

```text
控制面板
└── 设备和打印机
    └── 右键打印机
        └── 打印机属性
            └── 打印测试页
```

判断：

| 主机测试页结果 | 说明 | 下一步 |
|---|---|---|
| 可以打印 | 打印机本身大概率正常 | 检查共享和客户端 |
| 无法打印 | 问题在主机、驱动、端口或打印机 | 先修复主机打印 |
| 任务卡住 | 可能是队列或 Spooler | 清空队列并重启 Print Spooler |

---

## 6. 第三步：客户端访问共享主机

在客户端电脑按：

```text
Win + R
```

输入：

```text
\\共享主机电脑名
```

或：

```text
\\共享主机IP
```

例如：

```text
\\OFFICE-PC-01
\\192.168.0.25
```

判断：

| 访问结果 | 说明 |
|---|---|
| 能看到共享打印机 | 网络访问基本正常 |
| 提示输入账号密码 | 需要检查共享权限或凭据 |
| 无法访问 | 检查网络、防火墙、网络发现 |
| 找不到主机 | 检查主机名、IP、同网段 |

---

## 7. 第四步：检查网络发现与文件打印共享

在共享主机上检查：

```text
控制面板
└── 网络和共享中心
    └── 更改高级共享设置
```

建议启用：

```text
启用网络发现
启用文件和打印机共享
```

如果是公司网络，建议网络类型使用：

```text
专用网络
```

如果电脑当前被识别为“公用网络”，Windows 防火墙可能会阻止共享访问。

---

## 8. 第五步：检查打印机共享设置

在共享主机上：

```text
控制面板
└── 设备和打印机
    └── 右键打印机
        └── 打印机属性
            └── 共享
```

确认：

- 勾选“共享这台打印机”
- 共享名称简短、清晰，不建议使用特殊符号
- 必要时勾选“在客户端计算机上呈现打印作业”

推荐共享名示例：

```text
HP_M1216_OFFICE
Canon_2F_Admin
Brother_Warehouse
```

---

## 9. 第六步：处理访问被拒绝

如果客户端提示：

```text
访问被拒绝
无法连接到打印机
需要凭据
```

可以检查：

- 共享主机是否允许当前用户访问
- 客户端是否保存了错误凭据
- 主机和客户端是否在同一工作组或域环境
- 是否启用了密码保护共享

清理 Windows 凭据：

```text
控制面板
└── 凭据管理器
    └── Windows 凭据
```

删除旧的共享主机凭据后重新连接。

---

## 10. 第七步：检查 Windows 防火墙

共享打印机依赖文件和打印机共享相关规则。

在共享主机上检查：

```text
控制面板
└── Windows Defender 防火墙
    └── 允许应用或功能通过 Windows Defender 防火墙
```

确认允许：

```text
文件和打印机共享
网络发现
```

如果公司使用第三方杀毒或终端安全软件，也需要检查是否阻止局域网共享访问。

---

## 11. 第八步：驱动匹配问题

共享打印机常见驱动问题：

- 主机是 64 位系统，客户端也是 64 位，通常较简单
- 老旧打印机驱动不兼容新系统
- 客户端自动安装驱动失败
- 主机驱动损坏导致所有客户端异常

建议：

```text
优先在共享主机安装官方驱动
客户端连接失败时，先在客户端本机安装同型号官方驱动
再连接共享打印机
```

---

## 12. 重新连接共享打印机

客户端可以先删除旧共享打印机：

```text
控制面板
└── 设备和打印机
    └── 删除旧打印机
```

然后重新连接：

```text
Win + R
↓
\\共享主机IP
↓
双击共享打印机
```

也可以通过命令添加：

```cmd
rundll32 printui.dll,PrintUIEntry /in /n "\\共享主机名\共享打印机名"
```

示例：

```cmd
rundll32 printui.dll,PrintUIEntry /in /n "\\OFFICE-PC-01\HP_M1216_OFFICE"
```

---

## 13. 共享打印机速查表

| 故障现象 | 优先检查 | 常见原因 | 处理建议 |
|---|---|---|---|
| 所有客户端不能打印 | 共享主机 | 主机关机、睡眠、断网 | 开机并关闭睡眠 |
| 主机能打印，客户端不能打印 | 共享访问 | 权限、防火墙、凭据 | 检查共享和凭据 |
| 显示脱机 | 主机或网络 | 主机不可达、共享断开 | 访问 `\\主机名` 测试 |
| 提示访问被拒绝 | 权限 | 凭据错误、共享权限不足 | 清理凭据并重新连接 |
| 提示需要驱动 | 驱动 | 客户端缺少驱动 | 安装官方驱动 |
| 任务卡在队列 | Spooler | 主机队列异常 | 重启主机 Spooler |

---

## 14. 企业环境建议

如果一台打印机需要多人长期使用，建议逐步从共享打印机改为：

```text
网络打印机
    ↓
固定 IP / DHCP 保留
    ↓
Standard TCP/IP Port
    ↓
统一官方驱动
    ↓
统一设备登记表
```

共享打印机适合临时使用，但不建议作为企业长期部署方案。
