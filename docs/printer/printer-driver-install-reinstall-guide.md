# 打印机驱动安装、卸载与重装指南

> 适用于 Windows 办公环境中打印机无法添加、驱动异常、功能缺失、打印格式错乱、扫描不可用等问题。

---

## 适用场景

当出现以下情况时，通常需要检查或重装打印机驱动：

- 打印机可以被发现，但无法正常打印
- Windows 测试页可以打印，但 Word、Excel、PDF 打印异常
- 打印选项缺失，例如双面打印、纸盒选择、扫描功能不可用
- 打印内容格式错乱、字体异常、页面比例不对
- 添加打印机时提示驱动不可用
- 更换打印机端口后仍然无法打印
- 升级 Windows 或迁移电脑后打印异常

---

## 驱动异常的常见表现

| 现象 | 可能原因 | 处理方向 |
|---|---|---|
| 显示“驱动程序不可用” | 驱动未安装或损坏 | 重新安装官方驱动 |
| 只有基础打印功能 | 使用了 Windows 自动驱动 | 安装完整功能驱动 |
| 打印乱码 | 驱动语言不匹配或驱动损坏 | 换用 PCL / PS / 官方驱动 |
| 打印格式错乱 | 驱动与应用程序渲染不匹配 | 更新驱动或更换打印模式 |
| 双面打印选项消失 | 驱动未识别设备功能 | 安装完整驱动并检查设备设置 |
| 扫描不可用 | 只安装了打印驱动 | 安装完整功能套件 |

---

## 官方驱动、通用驱动与 Windows 自动驱动

| 驱动类型 | 优点 | 缺点 | 推荐场景 |
|---|---|---|---|
| 官方完整驱动 | 功能完整，兼容性最好 | 安装包较大 | 办公室常用打印机 |
| 官方基础驱动 | 稳定，体积较小 | 扫描等功能可能缺失 | 只需要打印功能 |
| 通用驱动 | 适合批量部署 | 部分机型功能不完整 | 企业多型号统一管理 |
| Windows 自动驱动 | 安装方便 | 功能少，稳定性不一定好 | 临时测试 |

企业环境建议优先使用厂商官方驱动，尤其是需要双面打印、扫描、纸盒选择、装订、权限打印等高级功能时。

## 安装前准备

记录以下信息：

```text
打印机品牌：
打印机型号：
连接方式：USB / 网络 / 共享打印机
打印机 IP：
当前端口：WSD / TCP/IP / USB / 共享路径
操作系统版本：Windows 10 / Windows 11
是否需要扫描功能：是 / 否
```

如果是网络打印机，建议先确认：

```cmd
ping 打印机IP
```

如果 Ping 不通，应先排查网络、IP 和端口问题，再重装驱动。

## 标准安装流程

### 方式一：通过 Windows 添加打印机

```text
设置
└── 蓝牙和设备
    └── 打印机和扫描仪
        └── 添加设备
```

如果 Windows 自动发现打印机，可以先添加测试。但企业环境中更推荐手动指定 TCP/IP 端口。

### 方式二：通过 TCP/IP 地址添加网络打印机

```text
控制面板
└── 设备和打印机
    └── 添加打印机
        └── 我需要的打印机不在列表中
            └── 使用 TCP/IP 地址或主机名添加打印机
```

输入打印机 IP，例如：

```text
192.168.0.193
```

端口类型建议使用：

```text
Standard TCP/IP Port
```

### 方式三：使用厂商安装程序

适合需要完整功能的设备，例如打印 + 扫描一体机。

```text
下载官方驱动
↓
运行安装程序
↓
选择网络连接或 USB 连接
↓
按向导完成安装
↓
打印测试页
```

## 彻底重装打印机驱动流程

当普通重装无效时，可以按下面流程做“干净重装”。

### 第一步：删除打印机设备

```text
控制面板
└── 设备和打印机
    └── 右键目标打印机
        └── 删除设备
```

或：

```text
设置
└── 蓝牙和设备
    └── 打印机和扫描仪
        └── 选择打印机
            └── 删除
```

### 第二步：停止 Print Spooler

```cmd
net stop spooler
```

### 第三步：清空打印缓存

```cmd
 del /Q /F %systemroot%\System32\spool\PRINTERS\*.*
```

如果提示权限不足，请使用管理员身份运行命令提示符。

### 第四步：启动 Print Spooler

```cmd
net start spooler
```

### 第五步：删除旧驱动包

```text
控制面板
└── 设备和打印机
    └── 打印服务器属性
        └── 驱动程序
            └── 选择旧驱动
                └── 删除
```

建议优先选择：

```text
删除驱动程序和驱动程序包
```

如果提示驱动正在使用，通常需要：

- 确认相关打印机已经删除
- 重启 Print Spooler
- 重启电脑后再次删除

### 第六步：重新安装官方驱动

从厂商官网选择对应型号和系统版本，重新安装驱动。

## PCL、PS、UFR、XPS 简单说明

| 驱动语言 | 说明 | 常见场景 |
|---|---|---|
| PCL | 通用性强，速度快 | 办公文档、Word、Excel |
| PS / PostScript | 图形和排版更稳定 | PDF、设计文件、复杂图形 |
| UFR / UFR II | Canon 常见驱动语言 | Canon 打印机 |
| XPS | Windows 图形打印路径 | 特定应用或新系统环境 |

如果普通文档打印正常，但 PDF、图纸、设计文件经常出错，可以尝试更换 PCL / PS 驱动进行对比测试。

---

## 驱动安装后的验证

安装完成后，建议按下面顺序验证：

```text
打印 Windows 测试页
↓
用记事本打印简单文本
↓
用 Word 打印普通文档
↓
用 PDF 阅读器打印 PDF
↓
检查双面打印、纸盒、扫描等功能
```

如果 Windows 测试页正常，但某个软件无法打印，问题可能不在驱动本身，而在软件打印设置或文件内容。

---

## 企业环境建议

- 统一使用官方驱动或经过测试的通用驱动
- 同型号打印机尽量使用同一驱动版本
- 网络打印机优先使用 Standard TCP/IP Port
- 避免不同电脑使用不同端口类型连接同一台打印机
- 建立打印机型号、IP、驱动版本、安装包路径记录
- 驱动升级前先在少量电脑测试

---

## 驱动问题速查表

| 问题 | 优先检查 | 推荐处理 |
|---|---|---|
| 驱动程序不可用 | 驱动是否损坏 | 删除并重装官方驱动 |
| 双面打印不见了 | 驱动是否完整 | 安装完整功能驱动 |
| 扫描不可用 | 是否安装扫描组件 | 安装完整套件 |
| 打印格式错乱 | 驱动语言 / 软件渲染 | 尝试 PCL / PS 驱动 |
| 添加打印机失败 | 端口 / 权限 / 驱动包 | 使用管理员权限安装 |
| 重装后仍异常 | 残留旧驱动 | 删除驱动包后干净重装 |

---

## 小结

打印机驱动问题经常和端口、队列、缓存、软件设置混在一起。排查时建议先确认网络与端口，再处理队列和 Print Spooler，最后再做驱动重装。对于企业环境，稳定的端口配置和统一驱动版本比单台电脑临时修复更重要。
